# Shelf Scout

A self-hosted PWA for tracking food expiration dates across a shared household inventory, with alerts and reports to reduce food waste.

## Language

### People and groups

**User**:
A person who has authenticated via Google and has an account in the system. A User may belong to more than one Household.
_Avoid_: Account, member, person

**Household**:
A named group that owns a shared food inventory. All Items, Locations, and settings belong to a Household. There is no concept of an individual resource within a Household.
_Avoid_: Home, family, group, team

**Membership**:
The association between a User and a Household, carrying a role of either admin or member. Admins can manage Locations, settings, and Invite Tokens; members can manage Items.
_Avoid_: Role, access

### Inventory

**Item**:
A single physical food instance stored in a Location with a specific expiration date. Each physical unit is its own Item record — two cans of the same product with different expiration dates are two Items. Items are tracked by presence only; quantity within an Item is not modeled.
_Avoid_: Food, product, entry, log entry

**Meal**:
A prepared or pre-cooked food stored as a unit (e.g., a container of leftovers, a batch of meal-prepped portions). Modeled as an Item — no recipe or ingredient composition is tracked.
_Avoid_: Recipe, dish, composed item

**Location**:
A named storage area within a Household (e.g., fridge, freezer, pantry). Locations are defined per Household and pre-seeded with fridge, freezer, and pantry on creation.
_Avoid_: Storage, zone, area

### Item lifecycle

**Consumed**:
An Item that has been removed from inventory because it was used or eaten. Recorded by setting `consumed_at`. The opposite of Expired.
_Avoid_: Used, eaten, removed

**Expired**:
An Item whose expiration date has passed and that has not been Consumed. The opposite of Consumed.
_Avoid_: Wasted, overdue, past-date

**Expiring Soon**:
An Item whose expiration date falls within the Household's configured Alert Threshold. The primary trigger for push notifications and dashboard alerts.
_Avoid_: Almost expired, near expiry

**Bulk Consume**:
The action of marking multiple Items as Consumed in a single operation. Used when a User cooks with several ingredients at once.
_Avoid_: Multi-remove, batch delete

### Access and notifications

**Invite Token**:
A time-limited, single-use credential generated by a Household admin. Sharing the token URL is the only way for a new User to join a Household.
_Avoid_: Invite link, invite code, referral

**Alert Threshold**:
The per-Household configuration of how many days before expiration a push notification should be sent. Defaults are 7 days for freezer Items and 3 days for fridge and pantry Items.
_Avoid_: Notification window, warning period, reminder lead time

**Push Subscription**:
A device-level registration for Web Push notifications, associated with a User and a Household. A User may have multiple Push Subscriptions across devices.
_Avoid_: Device token, notification endpoint
